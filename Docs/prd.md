# Product Requirements Document (PRD)

**Project:** Rarebird Infra Website
**Version:** 1.0 (Initial Skeleton Build)
**Prepared by:** Archit Mishra

---

## 1. Project Overview

Rarebird Infra is a real estate company focused on **developing and marketing township projects**.

This website will serve as a **brand and project showcase platform**, designed to:

* Present township developments
* Communicate credibility and scale
* Capture user enquiries

This version is a **design-aligned skeleton build**, intended for internal review and client validation before final development.

---

## 2. Scope of This Phase

### Included:

* Complete website structure
* All primary pages and sections
* Placeholder content and imagery
* Functional navigation
* Demo enquiry forms (UI only)

### Excluded:

* Backend integration
* Data storage or API connections
* CMS or dynamic admin panel
* Search or filtering systems
* Authentication features

---

## 3. Core Positioning

The website must reflect the following:

* Rarebird Infra is **not a property listing platform**
* Rarebird Infra does **not aggregate or search third-party listings**
* The platform is focused exclusively on:

  * Owned township projects
  * Affiliated developments
  * Brand-led storytelling

---

## 4. Website Architecture

### 4.1 Pages

The following pages are required:

1. Homepage
2. Township Detail Page (Template) – 2 instances
3. Blog Listing Page
4. Blog Detail Page

---

## 5. Homepage Requirements

The homepage must follow a **structured, section-based layout** aligned with the design reference.

---

### 5.1 Hero Section

**Purpose:** Establish brand tone and visual impact

**Requirements:**

* Full-width background image
* Large headline (placeholder content)
* Supporting text
* Primary CTA button

**Note:**
No search or filtering UI should be included.

---

### 5.2 About Section

**Purpose:** Introduce the company

**Requirements:**

* Section heading
* Two short descriptive paragraphs
* Supporting statistics displayed visually

---

### 5.3 Statistics Section

**Purpose:** Build credibility

**Requirements:**

* 3–4 key metrics
* Displayed in visually distinct containers (e.g., circular blocks)
* Example placeholders:

  * Projects Delivered
  * Acres Developed
  * Years of Experience

---

### 5.4 Services Section

**Purpose:** Communicate business offerings

**Requirements:**

* 3–4 service cards
* Each includes:

  * Image
  * Title
  * Short description

---

### 5.5 Township Showcase Section

**Purpose:** Highlight key projects

**Requirements:**

* Section title: “Our Townships”
* Display 2 township cards
* Each card includes:

  * Image
  * Project name
  * Location
  * Short description
  * CTA → Detail page

---

### 5.6 Visual Showcase Section

**Purpose:** Communicate scale and quality

**Requirements:**

* Large, high-quality image (e.g., aerial township view)
* Optional non-functional overlays
* No interactive map or pricing system

---

### 5.7 Blog Section

**Purpose:** Provide content structure for future use

**Requirements:**

* Section title
* 3–4 blog entries (placeholder content)
* Each includes:

  * Title
  * Date
  * Short excerpt

---

### 5.8 Call-to-Action Section

**Purpose:** Encourage user enquiries

**Requirements:**

* Headline text
* CTA button linking to enquiry interaction

---

### 5.9 Footer

**Requirements:**

* Logo (placeholder)
* Navigation links
* Contact details (placeholder)

---

## 6. Township Detail Page Requirements

A reusable template must be created for township pages.

---

### 6.1 Hero Section

* Large image
* Project name
* Location
* Tagline

---

### 6.2 Overview Section

* Detailed description
* Key highlights (bullet points)

---

### 6.3 Image Gallery

* Grid layout
* Multiple images (placeholder)

---

### 6.4 Project Details Section

* Plot sizes
* Pricing (placeholder)
* Amenities
* Location advantages

---

### 6.5 Enquiry Form

**Fields:**

* Name
* Phone
* Email
* Message

**Behavior:**

* No backend integration
* On submit → display success message

---

## 7. Blog Pages

### 7.1 Blog Listing Page

* List all blog entries
* Placeholder content

### 7.2 Blog Detail Page

* Title
* Cover image
* Full placeholder text

---

## 8. Content Strategy (Initial Phase)

All content will be placeholder-based:

* Text → Lorem Ipsum or generic copy
* Images → Stock/placeholder images
* Township names → Temporary placeholders

---

## 9. Scalability Considerations

The implementation should allow:

* Easy addition of new township pages
* Replacement of placeholder content
* Expansion of blog content

---

## 10. Acceptance Criteria

The build will be considered complete when:

* All pages are implemented and accessible
* Navigation flows correctly between pages
* Layout aligns with design reference
* Placeholder content is consistently applied
* Enquiry form displays success state
* No broken links or UI inconsistencies
